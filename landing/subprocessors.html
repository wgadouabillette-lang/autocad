Subprocessors

Last updated: June 15, 2026

Lyte uses the third-party service providers listed below (“Subprocessors”) to host, operate, and deliver the Service. We require Subprocessors that handle personal data to provide appropriate contractual protections and to process data only on our instructions, consistent with our Privacy Policy.

We may update this list from time to time. Material changes will be reflected on this page with an updated “Last updated” date.

Core infrastructure

Subprocessor Purpose Typical data Location
Google LLC (Firebase / Google Cloud) Authentication, database (Firestore), file storage, Cloud Functions, website hosting, OAuth sign-in Account identifiers, profile data, workspace and chat content, presence, signaling metadata, user preferences, billing flags, encrypted API key storage Firestore primary region: EU (eur3); Cloud Functions: United States (us-central1)
Stripe, Inc. Subscription billing, checkout, customer portal, payment webhooks Email, customer and subscription IDs, payment status, invoice metadata (Stripe processes card data directly; Lyte does not store full card numbers) United States / global
Vercel Inc. Application and API hosting (web app and backend endpoints) Request metadata, IP address, authentication tokens in transit, API payloads necessary to operate features United States / global edge network

AI providers (Pro plan and user-configured keys)

When you use AI features (chat assistant, AI Notes, follow-ups, vision/CAD assistance), message content and related context you submit may be sent to the model provider you or Lyte selects. If you provide your own API keys, requests are routed using those keys.

Subprocessor Purpose Typical data Location
OpenAI, LLC Large language model inference (e.g. GPT models) Prompts, chat history, optional images or documents you attach United States
Anthropic, PBC Large language model inference (Claude models) Prompts, chat history, optional attachments United States
xAI Corp. Large language model inference (Grok models) Prompts, chat history, optional images for vision features United States

Realtime communication

Subprocessor Purpose Typical data Location
Google LLC STUN servers for WebRTC connectivity (stun.l.google.com) IP addresses and connection metadata during voice/video calls Global
Metered.ca (Open Relay Project) TURN relay when direct peer connection is unavailable Encrypted media relay metadata; media may pass through relay servers when NAT traversal requires it Canada / global

Voice and video streams are primarily peer-to-peer via WebRTC. Lyte uses Firebase to exchange signaling data (session descriptions and ICE candidates), not to store call recordings unless you explicitly start a recording or AI Notes capture.

Identity and integrations

Subprocessor Purpose Typical data When used
Google LLC Sign in with Google; Google Calendar and Gmail chat connectors OAuth tokens, profile email/name, connector content you invoke via slash commands When you sign in with Google or use Google connectors
Microsoft Corporation Sign in with Microsoft; Outlook connector OAuth tokens, profile data, connector content you invoke When you sign in with Microsoft or use Outlook connector
Meta Platforms, Inc. Sign in with Facebook Facebook user identifier, email, name and public profile if shared When you sign in with Facebook

Other service providers

Subprocessor Purpose Typical data
Google Fonts (Google LLC) Delivery of the Inter typeface IP address and browser metadata sent to the font CDN

Optional TURN provider

If configured for your deployment, Lyte may use a dedicated TURN service (VITE_TURN_URL) instead of or in addition to the public relay above. That provider would process connection metadata and relay encrypted media when required.
